## 2.5.1-1 — Convert Sections A, B, C, D, G, H to `<details>`

**File:** `scorer.html`

Each of these sections needs to become a `<details open>` (open by default — matches the v2.2 audit) with a matching `<summary>` that preserves the existing H2 title and any inline status chip.

**Pattern to apply:** for every section listed below, wrap the existing inner content in `<details open id="..." class="..."><summary>…existing H2 block…</summary><div class="…original padding…">…existing body…</div></details>`.

**The `<summary>` MUST:**
- Carry the existing `step-title` H2 (with its step-badge letter).
- Include any status chip currently inside the H2 (rubric, feedback, cohort chips all sit inside their H2 — move the whole H2 into the summary).
- Be styled so the default disclosure triangle is removed (`list-none` + `::-webkit-details-marker { display: none; }` class pattern) — matches how F currently renders.
- Be clickable on the whole row.

**The `<details>` MUST keep the current section's:**
- `id` attribute (`sec-student`, `sec-rubric`, `sec-adjust`, `sec-feedback`, `sec-export`, `sec-cohort`).
- `class` for outer card styling (white card, shadow, border, rounded). Move the `p-5` / `p-4` off the outer `<details>` and into an inner `<div>` that wraps the body content — `<details>` itself should not have padding (otherwise collapsed sections leave a thick empty strip).
- `no-print` modifier where it currently exists (sections C, G, H).

**Sections to convert:**

- `sec-notes` (E) — already a `<details>`.

Wrap the body of each converted section in `<div class="section-body">` (or `section-body compact` for `sec-rubric`, which uses a full-width `<table>` and shouldn't get extra horizontal padding).

## Risks to watch

- **Form submit / default actions.** `<details>` inside a form can interfere if the scorer is inside a `<form>` — it isn't, but Roo should double-check none of the grade dropdowns get detached from their event handlers after the wrap.
- **`id` collisions.** Two sections currently have double `id` attributes in the HTML (`sec-adjust` plus `penalty-section`; `sec-cohort` plus `cohort-section`). Preserve both — move them onto the new `<details>` element. Second `id` attributes on the same element are invalid HTML; if Roo sees this, pick the functional one (`penalty-section`, `cohort-section`) and drop the v2.2 anchor `id`, then add the v2.2 anchor `id` onto an inner span at the top of the section body so the rail link still scrolls correctly.
- **Focus/scroll on rail click.** Clicking a rail link should scroll to the section AND open it if collapsed. Add this one-liner inside the rail script:

```js
document.querySelectorAll('#section-rail [data-rail]').forEach(function (a) {
  a.addEventListener('click', function () {
    const id = a.getAttribute('href').slice(1);
    const d = document.getElementById(id);
    if (d && d.tagName === 'DETAILS' && !d.open) d.open = true;
  });
});
```
